Deliverable 02 — B2B Order Placed event

One custom event, fired every time a B2B order is placed. This is what lets us measure whether a B2B inquiry converts.

Event spec

Event name
B2B Order Placed
Triggers on
Pipedrive deal moved to Won stage. (Post-self-customiser launch, also fires on Shopify orders tagged b2b_custom.)
Fires into
Klaviyo custom event endpoint, against the profile keyed by email.
Frequency
One event per deal Won. Reorders fire a fresh event each time.
Purpose
Conversion attribution. Match B2B Order Placed against the inquiry event to compute conversion rate by cohort, vessel, lead source, and use case.

Event payload

Each event carries the deal context inline so flow logic can read it without a second lookup. Suggested payload:

{
  "event": "B2B Order Placed",
  "customer_properties": { "$email": "buyer@example.com" },
  "properties": {
    "pipedrive_deal_id": "12345",    // deduplication key
    "deal_value": 4800.00,
    "currency": "AUD",
    "vessel_type": "Can",
    "quantity": 240,
    "custom_design": true,
    "order_date": "2026-05-30",
    "expected_delivery_date": "2026-06-13",
    "company_name": "Example Co",
    "account_manager": "Jay",
    "is_reorder": false    // true if this profile has a prior B2B Order Placed
  }
}
Why is_reorder matters: the reorder flow needs to know whether to treat an order as a first conversion or the start of a new cycle. Easiest place to set it is at the source, in the Pipedrive → Klaviyo handler, by checking if the profile already has a prior B2B Order Placed event.
