Implementation notes for Klaviyo setup
TriggerKlaviyo metric Filled Out Form, created natively by the Kickoff Labs integration. Do not use list-subscribe as trigger — waits for Social ID sync.
Smart sendingTurn OFF for this flow. Pre-launch warm-up needs every subscriber to receive every email for domain reputation. Revisit post-launch.
First-name fallbackUse {{ first_name|default:'mate' }} across all emails. "Mate" is the approved fallback per voice guide.
Share link merge tagEvery email's tertiary footer block uses: https://steakking.co.uk?kid={{ person|lookup:'Kickoff Labs Social ID'|default:'' }}
Sender addressJohnny-signed emails (E1, E2, E3, E6, E8, E9, E11) from johnny@steakking.co.uk. Brand-voice emails (E4, E5, E7, E10) from hello@steakking.co.uk.
Subject A/B testE1 has three subject line variants. Set up as Klaviyo A/B subject test with even split.
Domain warm-upCritical first 2-4 weeks. Do not run campaigns to this list until flow opens are stable. Monitor bounce rate daily.
Kickoff Labs ZapierNot required for v1 of this flow. All personalisation uses native properties. Zapier only needed if later adding dynamic referral count / tier content.
