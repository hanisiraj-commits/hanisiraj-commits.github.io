📋 Klaviyo setup checklist for implementation
  • Trigger: Metric → Referral Made (provisioned by KickoffLabs integration)
  • Trigger filter: campaign equals Steak King UK Pre-Launch
  • Flow filter (profile-level): Subscribed to email · AND · has not unsubscribed in last 30 days
  • Smart sending: OFF — proximity moments must land regardless of recent sends
  • Re-entry: ALLOWED on every Referral Made event
  • Time delays: NONE — all sends immediate on event fire
  • Conditional split: 5 branches. Evaluate event.referral_count equals N for each. The else branch ends the flow with no send.
  • Branch order: 4 → 9 → 24 → 40 → else
  • Personalisation tokens (every email): {{ event.social_url }} · {{ event.social_id }} · {{ event.referral_count }} for the "N of tier" progress label
  • Templates: Pull all four email designs from Figma node 416-494 — E1 = frame 416:136 · E2 = 416:689 · E3 = 416:1238 · E4 = 416:1760
  • Subject/Preview frames are separate siblings — E1 416:124 · E2 416:127 · E3 416:130 · E4 416:133 (load these into Klaviyo subject + preview fields)
  • This flow only sends proximity nudges — there are no celebration/unlock emails when a tier is hit (count == 5, 10, 25, 50). Those exit silently from this flow. Confirm with Hani whether a separate unlock flow is needed.
  • QA test plan: Fire mock events at counts 4, 9, 24, 40 → confirm each routes to the correct email. Fire at counts 5, 10, 25, 50, 7, 12 → confirm silent exit (no email sent).
