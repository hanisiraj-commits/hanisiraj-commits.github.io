📋 Klaviyo setup checklist for implementation
Why two flows? Shopify's webhook-based Added to Cart only fires when the shopper is identified server-side. Klaviyo's onsite JS catches anonymous visitors who've been cookie-matched via prior email engagement. Running both maximises coverage. The Flow B suppression filter prevents profiles who already received Flow A from getting a duplicate sequence.
Flow A — F_CART-ABANDONMENT_SHOPIFY (build first)
  • Trigger: Metric → Added to Cart · source = Shopify integration. Confirm in Klaviyo Metrics panel that the metric's integration column shows "Shopify".
  • Trigger filter (event): $value > $0.
  • Flow filter (profile-level) — apply all four:
    • What someone has done → Placed Order zero times since starting this flow.
    • What someone has done → Started Checkout zero times since starting this flow.
    • Has not been in this flow in the last 14 days.
    • Can receive email marketing = True.
  • Smart sending: ON.
  • Re-entry: ALLOW.
  • Delays: 2hr → E1 · 1 day → E2 · 1 day → E3.
  • Conditional split (post-E1): Placed Order equals 0 over all time → YES = NEW · NO = REPEAT.
Flow B — F_CART-ABANDONMENT_KLAVIYO (clone Flow A, then modify)
  • Clone Flow A (Flow → ⋯ → Clone). Rename to F_CART-ABANDONMENT_KLAVIYO.
  • Change trigger source: Metric → Added to Cart · source = Klaviyo onsite tracking (NOT Shopify). Confirm via integration column.
  • Required: Active on Site tracking — Klaviyo's klaviyo.js snippet must be installed in the Shopify theme for this metric to fire reliably.
  • Increase initial delay: 2 hours → 4 hours. Gives Flow A first crack at the profile.
  • Add suppression filter: Received Email zero times in last 30 days where Flow equals F_CART-ABANDONMENT_SHOPIFY.
  • Everything else (split, branches, templates, content) stays identical to Flow A.
Shared setup details
  • Personalisation tokens: {{ event.extra.line_items }} for cart product block (test for both ATC payload shapes) · {{ person.first_name|default:'there' }} in greeting.
  • Discount code: CART20 for NEW branch. Confirm with Andy whether static or Klaviyo-generated unique code.
  • Templates: Figma file fpOouAaKYOInN4Fg9WDUBs. Map to BON CHARGE canonical 10-module structure.
  • QA test plan:
    • Fire Shopify ATC on test profile with Placed Order = 0 → enters Flow A NEW, E2 contains CART20.
    • Fire Klaviyo onsite ATC on a different test profile → enters Flow B, 4hr delay applied.
    • Fire BOTH events on the same profile → Flow A sends; Flow B should be blocked by the dedup filter.
    • Mid-flow purchase: enter Flow A, complete checkout before E2 → confirms profile exits.
    • Empty cart test → trigger filter blocks entry.
