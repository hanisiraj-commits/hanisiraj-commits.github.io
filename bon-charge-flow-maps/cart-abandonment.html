Email 1 — 2hr reminder (single send, no split)
SUBJECT: "Your cart's still warm" PREVIEW: "Your items are saved — finish whenever you're ready."
Angle
Gentle, brand-led nudge. No discount, no urgency. Frames the cart as held safely so the shopper feels no pressure but is reminded the items are still there.
Modules
  • Dynamic cart product block — {{ event.extra.line_items }}
  • One-line value reinforcement (Andy voice — science-led, no claims tells)
  • Single CTA — RETURN TO CHECKOUT (deep link via {{ event.extra.checkout_url }})
  • Footer per BON CHARGE design system (sans wordmark, generic SHOP NOW colour rules)

📋 Klaviyo setup checklist for implementation
  • Trigger: Metric → Checkout Started (Shopify integration). Klaviyo officially recommends Checkout Started over Added to Cart because the shopper has already submitted an email — higher intent, fewer false fires.
  • Trigger filter (event): $value greater than $0 — protects against empty / test carts. Optionally raise to a margin floor (e.g. $value > $50) if Andy wants to limit incentives to higher-AOV carts.
  • Flow filter (profile-level) — apply all three:
    • What someone has done (or not done) → Placed Order zero times since starting this flow — the canonical conversion-suppression filter. Removes anyone who completes purchase mid-flow.
    • Has not been in this flow in the last 30 days — agency-standard window for Checkout Started (vs 14 days for Added to Cart). Protects margin and prevents discount-hunting on the NEW branch.
    • Can receive email marketing = True — explicit consent check.
  • "Checkout still open" not natively supported. Klaviyo doesn't expose a cart-status field — the Placed Order zero times since starting this flow filter is the canonical proxy.
  • Smart sending: OFF for this flow. Agency consensus — the 16hr quiet window blocks legitimate high-intent sends. (Klaviyo's general default is ON; we're overriding for cart abandonment specifically.)
  • Re-entry: ALLOW re-entry. Checkout Started fires per session; the 30-day flow filter handles throttling.
  • Time delays: 2 hours → Email 1 · 1 day → Email 2 · 1 day → Email 3. Matches current Klaviyo + agency best practice (BS&Co, SyncSpark 2026).
  • Conditional split (post-E1): What someone has done → Placed Order equals 0 over all time → YES routes to NEW branch · NO routes to REPEAT branch. Critical — must be a conditional split, not a flow filter (a flow-level filter here would suppress repeat buyers entirely).
  • Branch order: evaluate NEW (Placed Order = 0) first; everything else falls into REPEAT.
  • Discount code: single static code CART20 for NEW branch. Confirm with Andy whether this should be a unique Klaviyo-generated code per profile (preferred for tracking + abuse protection) or a static code.
  • Personalisation tokens: {{ event.extra.line_items }} for cart product block · {{ event.extra.checkout_url }} for the resume-checkout deep link · {{ person.first_name|default:'there' }} in greeting (BON CHARGE voice — keep it minimal, no "Hey [name]!" energy).
  • Templates: source from Figma file fpOouAaKYOInN4Fg9WDUBs. Map to BON CHARGE canonical 10-module structure (stats block replaces narrative opener, UGC mandatory, site-wide + HSA/FSA modules at footer).
  • BON CHARGE compliance pass: all copy must clear the Permissible Claims checklist before send (TGA-aligned banned words, em-dash rule for Andy voice, humanizer pre-pass).
  • QA test plan:
    • Fire a Checkout Started event on a test profile with Placed Order = 0 → confirms routing into NEW branch, E2 contains CART20.
    • Fire on test profile with Placed Order ≥ 1 → confirms routing into REPEAT branch, E2 contains no code.
    • Mid-flow purchase test: fire Checkout Started, place order manually within 24hr, confirm profile drops from flow before E2.
    • Re-fire Checkout Started inside the 30-day window → profile should NOT re-enter.
    • Empty cart test → trigger filter should block flow entry.
