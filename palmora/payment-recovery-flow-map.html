📋 Klaviyo setup checklist for implementation
  • Trigger: Metric — Subscription Renewal Failed (RWL4Qw, source API/Seal Subscriptions).
  • Smart sending: OFF — quasi-transactional, must always deliver.
  • Conditional gates: before E2 and E3, add a conditional split "what someone has done → Renewed Subscription (TKcJ8t) zero times since starting this flow." If they've renewed, exit (stop dunning immediately).
  • Time delays: E1 immediate · E2 +2 days · E3 +2 days (Day 4). Align E3 with Seal's final retry / auto-cancel timing so "we'll pause" is accurate.
  • Re-entry: allow re-entry (a future failed renewal should re-trigger).
  • No discount in this flow. No 30-day guarantee/refund language anywhere.
  • Personalisation tokens: {{ event.ShippingFirstName }} / {{ first_name }} · {{ scent }} (parse event.ItemNames after last " - ") · {{ event.SubscriptionMagicUrl }} · {{ event.$value }} · {{ next_order_date }} (Seal next charge/ship date; fallback "the 28th").
  • E3 plain text: send from founder name, real monitored reply-to.
  • Copy source: Flows Copy sheet cols H–J. QA test: fire a Renewal-Failed event → confirm E1 immediate; fire Renewed Subscription → confirm exit before E2.
