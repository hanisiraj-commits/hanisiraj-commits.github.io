📋 Klaviyo setup checklist for implementation
  • Trigger: Metric — Cancelled Subscription (RunePz, source API/Seal). Cancels are immediate (CancellationScheduledFor null).
  • Smart sending: 24h.
  • Exit gate: each step checks Started Subscription (XAsM5N) since flow start → if they reactivated, exit.
  • Conditional split (Day 4, before E3): "what someone has done → trigger event property ItemNames contains Palmora Box." YES → send E3 (upsize) then continue to E4. NO (else) → skip E3, go straight to E4. The else path must be wired explicitly so non-box cancellers still receive E4.
  • Time delays: E1 immediate · E2 +2 days · split +2 days (Day 4) · E3 Day 4 · E4 Day 8.
  • Win-back code: create COMEBACK15 — 15%, sitewide, stacks on subscription + one-time, 7-day expiry from send.
  • Confirm before launch: free-gift SKU (E2 {{ gift }}) · single full-size 35ml subscription product + subscribe-and-save price (E3) · {{ next_order_date }} mapping.
  • Reassurance ("pause or cancel anytime, never locked in") appears in every email. No 30-day guarantee/refund language.
  • Personalisation tokens: {{ first_name }} · {{ scent }} · {{ event.SubscriptionMagicUrl }} · {{ event.DiscountValue }} · {{ event.OrderIntervalFrequency }} + {{ event.OrderIntervalUnit }}.
  • Plain text (E4): founder name, real monitored reply-to. Copy source: Flows Copy cols K–N.
  • QA test: fire Cancelled Subscription with ItemNames "Palmora Box" → expect E1, E2, E3, E4. Fire one with a 35ml item → expect E1, E2, (skip E3), E4. Fire Started Subscription mid-flow → expect exit.
